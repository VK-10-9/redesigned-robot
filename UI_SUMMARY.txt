================================================================================
1. PROJECT OVERVIEW
================================================================================

MISSION:
Transform Aadhaar biometric and demographic data into actionable intelligence
for enrollment analysis, fraud detection, and policy optimization.

KEY STATISTICS:
- 6 Million+ enrollment records
- 28 States covered
- 40+ API Endpoints
- 6 Intelligence Frameworks
- 98% accuracy rate
- Real-time processing

FRAMEWORKS DEPLOYED:
1. ADIF (Aadhaar Data Integrity Framework) - Data normalization & deduplication
2. IRF (Identity Resilience Framework) - Biometric aging & escalation
3. AFIF (Aadhaar Forensic Intelligence Framework) - Fraud pattern detection
4. PROF (Program Resource Optimization Framework) - Demand forecasting
5. AMF (Aadhaar Mobility Management Framework) - Cross-state tracking
6. PPAF (Privacy-Preserving Analytics Framework) - Differential privacy

API CLIENT USAGE:

// GET requests
const data = await apiClient.get('/api/national-overview')

// GET with parameters
const filtered = await apiClient.get('/api/data', {
  params: { state: 'Karnataka', limit: 100 }
})

// POST requests
const result = await apiClient.post('/api/analysis', {
  state: 'Maharashtra',
  startDate: '2024-01-01'
})

// Error handling
try {
  const data = await apiClient.get('/api/endpoint')
} catch (error) {
  console.error('Error:', error.message)
}

20. CONTRIBUTING GUIDELINES
================================================================================

CODE STANDARDS:
1. Follow ESLint configuration rules
2. Use Prettier for code formatting
3. Write TypeScript (strict mode enabled)
4. Add unit tests for new features
5. Document complex logic with comments
6. Follow naming conventions (camelCase for JS, PascalCase for React)

GIT WORKFLOW:
1. Clone repository: git clone https://github.com/samvidhan/ui.git
2. Create branch: git checkout -b feature/your-feature-name
3. Make changes and test
4. Commit: git commit -m "Description of changes"
5. Push: git push origin feature/your-feature-name
6. Create Pull Request on GitHub
7. Wait for review and address feedback
8. Merge when approved

COMMIT MESSAGES:
Format: [TYPE] Brief description
Types: feat, fix, docs, style, refactor, test, chore

Examples:
[feat] Add data export functionality
[fix] Fix anomaly detection algorithm
[docs] Update API documentation
[style] Format code with Prettier

================================================================================
21. TROUBLESHOOTING & FAQ
================================================================================

COMMON ISSUES:

Issue: Port 3000 already in use
Solution: npm run dev -- -p 3001 (use different port)

Issue: TypeScript errors on build
Solution: npm run type-check (identify issues)

Issue: Styling not applying
Solution: npm run build (rebuild TailwindCSS)

Issue: API not responding
Solution: Check backend is running (npm run dev in backend)

Issue: Data not loading
Solution: Check network tab, verify API endpoints

PERFORMANCE ISSUES:
- Clear browser cache
- Disable browser extensions
- Check Network tab for slow requests
- Profile with DevTools Performance tab
